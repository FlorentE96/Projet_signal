MIR-QBSH: A Corpus for Designing QBSH (Query by Singing/Humming) Systems

Roger Jang, MIR Lab at NTHU

This directory contains QBSH (Query By Singing/Humming) corpus by MIR (Multimedia Information Retrieval) Lab at CS Dept of NTHU (National Tsing Hua University), Taiwan. The corpus can be used for designing QBSH systems which accept a user's singing/humming input to retrieve the intended songs. You can click here to download the corpus.

The corpus has been used for the QBSH task at MIREX (Music Information Retrieval Evaluation eXchange) 2006~2009.

The directory structure of the corpus is as follows:

  • midiFile: This folder contains 48 ground-truth MIDI files. List of these MIDI files is available at songList.txt, which is tab-separated file with 4 columns: file name, English title ('-' if none), Chinese title ('-' if none), number of wav file recordings.
  • waveFile: This folder contains 4431 singing/humming clips from about 195 subjects. Note that
    • Each subdirectory under the folder represents the recordings in a specific year. A list of these subdirectories is available at yearDirInfo.txt, which is tab-separated file with 4 columns: directory name, number of persons in a directory, number of wav files in a directory, original name of the directory (which contains big5-encoded traditional Chinese characters).
    • Each subdirectory under the "year" directory is the "person" directory, representing the recordings of each person in a specific year. The mapping of the "person" directory to their real names (in Chinese) is available at the file "personDirInfo.txt" within each "person" directory.
    • File extension within each "person" directory:
      • *.wav: Wav files of 8 KHz, 8 bit, mono.
      • *.pv: For each .wav file, there is a corresponding .pv (pitch vector) file. The .pv file contains manually labelled pitch file, with frame size = 256 and overlap = 0. We have tried our best to make the manually labeled pitch as correct as possible. However, since the pitch was labelled by the student who did the recording, there is no guarantee for their correctness. (Some of the early recordings do not have corresponding pv files.)
